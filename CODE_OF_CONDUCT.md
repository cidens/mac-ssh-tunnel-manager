# 贡献者行为准则

## 我们的承诺

为了营造开放、友善和互相尊重的社区环境，项目维护者和贡献者承诺：任何人都可以在不受年龄、体型、残障、族裔、性别特征、性别认同与表达、经验水平、教育程度、社会经济地位、国籍、个人外貌、种族、宗教或性取向影响的情况下参与本项目。

我们将以有助于建设开放、友好、多元、包容和健康社区的方式参与项目。

## 积极行为

有助于形成良好社区环境的行为包括：

- 对不同经验、观点和背景保持尊重。
- 提供建设性反馈，并愿意接受合理的技术意见。
- 对社区和用户负责，发现错误后及时澄清和修正。
- 优先考虑项目整体和用户安全，而不是个人立场。
- 对公开内容进行脱敏，不披露他人的主机、地址、账号、凭据或其他隐私信息。

不可接受的行为包括：

- 使用带有性暗示、侮辱、贬损、威胁或骚扰性质的语言和内容。
- 人身攻击、政治攻击、恶意挑衅或持续打断正常技术讨论。
- 未经允许公开他人的私人信息，包括地址、邮箱、账号、网络拓扑或安全配置。
- 诱导他人公开提交密码、token、私钥、完整 SSH 配置或其他敏感信息。
- 维护者合理判断为不适合专业开源社区的其他行为。

## 维护者责任

维护者负责解释和执行本行为准则，并可以对不当内容采取编辑、删除、锁定、拒绝或临时、永久限制参与等措施。

处理事件时，维护者应尽可能保护报告者和受影响人员的隐私，并说明处理决定的理由。维护者不得利用管理权限打击正常技术异议。

## 适用范围

本准则适用于仓库的 Issue、Pull Request、Discussion、代码审查、提交记录和其他代表本项目参与的公开或私密空间。

## 报告与执行

普通技术分歧应优先在对应 Issue 或 Pull Request 中基于事实讨论。

涉及骚扰、隐私、安全或不适合公开说明的行为，可以通过仓库的 [私密报告入口](https://github.com/cidens/mac-ssh-tunnel-manager/security/advisories/new) 联系维护者，并在标题中注明“行为准则报告”。不要在公开 Issue 中提交敏感证据。

这是个人维护的开源项目，不承诺固定响应 SLA；维护者会优先处理威胁用户安全、泄露敏感信息或持续破坏社区协作的问题。

## 处置原则

维护者会根据影响和重复程度采取递进措施：

1. **纠正**：私下或公开指出不当行为，要求停止并修正。
2. **警告**：对重复或较严重行为给出明确警告，并限制相关互动。
3. **临时限制**：在一定时间内限制参与 Issue、PR 或其他社区活动。
4. **永久限制**：对持续骚扰、威胁、安全滥用或严重侵犯隐私的参与者永久限制参与。

## 来源

本准则参考 [Contributor Covenant 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)，并根据本项目由个人维护、涉及 SSH 和网络配置的安全边界进行了调整。
