# 本地监听端口推荐验收记录

## 范围

- Issue：#23
- 开发分支：`codex/issue-23-local-port-recommendation`
- 应用版本：`0.4.0` 开发测试包，不是公开 Release

## 已实现功能

- 在规则编辑器的本地监听端口旁提供显式“推荐可用端口”入口，不自动改写草稿。
- 保存或启动因本地监听冲突失败时提供同一推荐入口；启动冲突会打开对应规则并自动发起推荐。
- 从当前端口的下一个非特权端口开始查找；到达 `65535` 后从 `1024` 回绕，并且不会把原端口重新推荐给用户。
- 每次请求只执行一次 `/usr/sbin/lsof -nP -iTCP -sTCP:LISTEN` 快照，同时排除其他连接组及当前草稿其他规则中的全部本地、动态 SOCKS 监听；禁用规则也作为应用内已配置端口排除。
- 通配地址与任意同端口地址冲突；`localhost` 与 `127.0.0.1`、`::1` 视为回环重叠；IPv4 与 IPv6 的具体非通配地址不直接合并。
- 推荐结果仅在用户点击“使用”后写入当前规则。其他规则不变。
- 仅当规则的 `openURL` Host 与旧监听地址重叠且显式端口等于旧监听端口时，同步 URL 端口；路径、查询和片段保持不变。
- 采用推荐端口后清除该规则原有的危险监听确认签名，后续按新端点重新确认。
- 推荐不代表端口预留；保存和启动继续执行既有的应用内冲突与系统监听复检。
- 监听地址为空或包含不支持字符时不执行 `lsof`；快照失败、超时、取消或无可用端口均返回明确错误，不猜测端口。
- `lsof`、监听结果解析、占用索引构建和候选搜索均在脱离主线程的任务中执行；编辑、删除、关闭窗口或继续修改规则会取消或废弃旧结果。
- 功能不增加持久化字段，不改变 JSON schema，也不终止或修改任何占用端口的进程。

## 性能验收方法

- 使用 Release 构建。
- 每项测试预热 5 次，正式记录 30 次。
- 记录中位数与 P95，以 P95 作为通过标准。
- 纯逻辑数据规模为 1000 个连接组、每组 20 条规则，共 20,000 个监听端点。
- 纯逻辑测试从 `65535` 开始回绕，覆盖占用端口索引构建和候选搜索。
- 端到端测试每次调用真实 `lsof`，并经过应用的异步推荐服务；不涉及 SSH 建连或公网请求。
- 主线程测试注入每次阻塞 75 ms 的假快照提供器；如果快照错误地在主线程执行，指标会超过 50 ms 门槛。

测试环境：

- 机型：MacBook Pro（Mac14,7）
- 芯片：Apple M2，8 核
- 内存：16 GB
- 系统：macOS 26.5（25F71）
- 应用版本：0.4.0 开发构建
- 测试日期：2026-07-20

执行命令：

```bash
env CLANG_MODULE_CACHE_PATH=/private/tmp/ssh-tunnel-manager-swift-cache \
  SWIFTPM_MODULECACHE_OVERRIDE=/private/tmp/ssh-tunnel-manager-swift-cache \
  swift test -c release --disable-sandbox \
  --filter 'slowPortSnapshot|localPortRecommendationEndToEnd|largeRecommendationIndex'
```

| 指标 | 中位数 | P95 | 门槛 | 结果 |
| --- | ---: | ---: | ---: | --- |
| 20,000 个监听端点的索引构建与回绕搜索 | 13.974 ms | 17.155 ms | 100 ms | 通过 |
| 真实 `lsof` 的端到端推荐 | 38.301 ms | 48.536 ms | 1,000 ms | 通过 |
| 注入 75 ms 慢快照时的主线程响应 | 6.306 ms | 6.519 ms | 50 ms | 通过 |

端到端测试每轮只执行一次 `lsof`；30 次正式样本对应 30 次快照，另有 5 次预热快照。外部网络、SSH 认证和目标服务响应不进入上述计时。

## 实机界面验收结果

验收使用无调试器的发布配置应用包，并在当时通过 `CFFIXED_USER_HOME` 尝试把配置、设置和通知数据隔离到 `/private/tmp`。验收前后真实 `tunnels.json` 的 SHA-256 保持一致，测试过程中没有改写或启动用户的真实隧道配置。

> 2026-07-20 后续复核：macOS 26.5 的 `FileManager` 不保证跟随 `CFFIXED_USER_HOME` 返回隔离的应用支持目录，因此原记录中“没有读取真实配置”的表述缺少可靠证据。Issue #21 已增加绝对路径限定的 `SSH_TUNNEL_MANAGER_APPLICATION_SUPPORT_DIRECTORY`，后续发布包验收统一使用该显式覆盖，避免把环境变量误当成隔离证明。

1. 通过：本地转发与动态 SOCKS 规则显示推荐按钮；展开远程转发规则后没有推荐按钮。
2. 通过：目标规则当前端口为 `18080`，当前草稿的禁用 SOCKS 规则占用 `18081`，另一禁用配置占用 `18082`，真实本机进程监听 `18083`；界面推荐结果为 `18084`，证明草稿、禁用配置和一次系统快照均进入排除集合。
3. 通过：推荐结果出现后，点击“使用”前端口仍为 `18080`，`openURL` 仍为 `http://127.0.0.1:18080/path?q=1#section`。
4. 通过：点击采用后只把当前规则改为 `18084`，匹配的 `openURL` 只更新端口，路径、查询和片段保持不变；SOCKS `18081`、远程规则本地目标 `3000` 及其他连接组均未变化。
5. 通过：推荐结果出现后由本机进程占用 `18084`，保存明确提示该端口已监听，没有写入磁盘；重新推荐跳过 `18084` 并得到 `18085`，保存成功。
6. 通过：本机进程监听 `18090` 时点击对应连接组“启动”，状态变为失败并显示具体规则和端点；错误区域的推荐入口自动打开对应规则并得到 `18091`，没有进入 SSH 建连。
7. 通过：得到英文建议 `Available port: 18086` 后继续修改草稿，旧建议立即消失；另有注入慢快照的自动化测试确认取消会传播到进行中的快照任务，不接收过期结果。
8. 通过：简体中文与英文主面板、编辑器、状态、失败提示和推荐文案均正确；推荐图标的辅助功能帮助分别为“推荐可用端口”和 `Recommend Available Port`。真实 `lsof` 在约 50 ms 内完成，加载态持续时间很短，但界面保持可交互，主线程性能测试和中英文资源测试均通过。
9. 通过：验收结束后发布测试包、三个临时监听进程均已停止，`18083`、`18084`、`18090` 均无遗留监听。
