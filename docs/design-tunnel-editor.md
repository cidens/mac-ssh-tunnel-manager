# 隧道编辑器 UI 设计

## 设计目标

将“快速查看与启停”和“复杂配置编辑”分离：菜单主面板保持紧凑稳定，新增或编辑作为一个独立、可取消的任务在 Sheet 中完成。

## 信息架构

### 主面板

- 保留搜索、标签、排序、状态、启动、停止、打开、编辑、详情和删除。
- 卡片不持有编辑草稿，也不内联渲染 `TunnelFormView`。
- “添加隧道”始终打开新建编辑器，不再承担“收起表单”状态。

### 编辑 Sheet

编辑器采用三段式布局：

1. 固定标题区：新增或编辑标题、配置名称说明。
2. 可滚动内容区：基本信息、转发规则、自动化。
3. 固定操作区：错误信息、取消、保存或更新。

建议初始尺寸为约 `680 × 700` 点；内容超过可用高度时只滚动中间区域。主面板本身不跟随编辑内容调整高度。

## 组件与状态

### `TunnelEditorRequest`

由 `TunnelMenuView` 持有的可选展示状态：

- 新建：不含既有配置，初始草稿为 `TunnelDraft()`。
- 编辑：包含目标 `TunnelConfig`，初始草稿由 `TunnelDraft(tunnel:)` 构建。

该状态作为 `.sheet(item:)` 的唯一来源，保证一次只显示一个编辑器。

### `TunnelEditorView`

职责：

- 持有当前草稿、初始草稿和展开规则 ID。
- 根据新建或编辑模式调用外部保存闭包。
- 判断草稿是否已修改。
- 管理取消确认、内部滚动和新增规则定位。
- 在自身 `ZStack` 中显示 `TunnelManager.riskWarning`，保证确认位于 Sheet 可见层级。

它不负责持久化、SSH 命令生成或进程控制。

### `TunnelFormView`

保留字段绑定和规则编辑能力，但移除主列表滚动回调和底部操作按钮。保存、取消与错误显示由 `TunnelEditorView` 的固定操作区统一管理。

### `TunnelRuleDraftView`

拆分为：

- 摘要头：展开按钮、启用开关、模式、端点摘要、移动和删除。
- 详情区：仅当该规则被选中时显示模式选择器、监听端、目标端和 URL。

展开状态属于编辑器，不写入 `TunnelDraft` 或配置文件。

### 自动化分组

“应用启动时连接”和“断线后自动重连”使用独立的浅色圆角卡片，与基础字段拉开层级。卡片顶部使用紧凑粗体标题，开关和辅助说明在卡片内统一缩进；连接组与 SSH Config 引用使用相同布局，避免自动化设置看起来像 Tags 后的一行普通字段。

## 状态流

### 新建

1. 主面板点击“添加隧道”。
2. 创建新建请求并展示 Sheet。
3. 用户编辑草稿；规则新增只影响 Sheet 内滚动。
4. 保存调用 `TunnelManager.addTunnel`。
5. 成功后关闭 Sheet，并由主列表滚动到新配置 UUID；失败则保留草稿并显示错误。

### 编辑

1. 卡片点击“编辑”，创建包含目标配置的请求。
2. Sheet 从持久化配置生成独立草稿。
3. 保存调用 `TunnelManager.updateTunnel`。
4. 若连接组正在运行，现有重启确认显示在 Sheet 上方；确认后保存、关闭并统一重启。
5. 失败或取消确认时 Sheet 与草稿保持。

### 取消

- `draft == initialDraft`：直接关闭。
- `draft != initialDraft`：显示放弃确认。
- 继续编辑：关闭确认并保留 Sheet。
- 放弃修改：关闭 Sheet，不调用 Manager 保存接口。

## 布局稳定性策略

- 主列表继续使用 `LazyVStack`，但所有行保持紧凑固定结构，不包含大型条件表单。
- 编辑器内部使用普通 `VStack`，最多 20 条摘要卡片，不依赖懒加载回收。
- 只有选中规则的详情区参与高度变化；不对整个 Sheet 或主列表执行大范围 `withAnimation`。
- 新增规则后通过编辑器自己的 `ScrollViewReader` 定位该规则，目标 ID 只存在于 Sheet 内。
- 保存与取消按钮不放进滚动内容，避免长表单把关键操作推到屏幕外。

## 确认层级

`TunnelManager.riskWarning` 同时服务以下场景：

- 危险本地或远程监听确认。
- 运行中编辑后的统一重启确认。

主面板启动操作仍由主面板覆盖层显示；编辑器发起的确认由 Sheet 内覆盖层显示。任意时刻只渲染用户当前可见层级中的一个确认，不允许确认视图留在 Sheet 后方。

## 错误与回滚

- `manager.addError` 在编辑器固定操作区上方显示。
- 保存失败不关闭 Sheet，不重建草稿。
- Manager 继续负责写盘失败回滚；编辑器不直接修改 `manager.tunnels`。
- 风险确认取消后不保存、不关闭 Sheet。

## 兼容性

- 不改变公开类型、JSON、配置目录或备份文件。
- 不改变 `TunnelDraft.makeConfig`、`SSHCommandBuilder` 和 `TunnelConfigStore` 的输入输出。
- 现有配置打开后使用相同 UUID、规则顺序、启用状态和风险确认签名。

## 测试设计

- 单元测试：`TunnelDraft` 可比较、规则摘要、展开选择和脏草稿判断。
- 视图状态测试：新建/编辑请求的初始值、保存成功关闭、保存失败保持、取消确认。
- 现有测试：配置校验、端口冲突、风险确认、运行中重启和本地化键一致性全部保留。
- 手工测试：中英文真实应用包，1、3、20 条规则，主列表滚动位置、Sheet 内滚动、风险确认层级和取消保护。

## 方案审查结论

### 已修复的高优先级缺口

- 风险确认若仍挂在主面板会被 Sheet 遮挡；设计明确在编辑器可见层级显示。
- 只移动表单但继续同时展开所有规则，仍会产生超长内容；设计限制为单规则展开。
- 取消直接关闭可能导致误丢数据；设计增加脏草稿确认。
- 把保存检查移到启动时会破坏既有可靠性契约；设计明确保持保存前拒绝。

### 残余风险

- SwiftUI Sheet 与自定义菜单 `NSPanel` 的焦点和 Escape 行为必须通过真实应用包验证，自动化测试不能单独证明。
- 编辑 Sheet 在显示期间通过菜单面板模态交互通知暂停外部点击关闭；消失后沿用短暂尾随事件抑制，避免保存或取消 Sheet 时同时关闭主面板。
- 20 条规则时摘要列表仍较长，但只在 Sheet 内滚动，不再影响主列表；若未来规则数量继续增长，再评估独立管理窗口。
