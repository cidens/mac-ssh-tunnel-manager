# SSH Config 只读导入验收记录

## 验收范围

本文记录 GitHub Issue #4“从 SSH Config 只读发现并导入转发配置”在 2026-07-18 完成的最终验收。

验收覆盖：

- 只读扫描默认 `~/.ssh/config` 及可访问的 `Include` 文件。
- 发现明确 Host、大小写不敏感去重和通配 Host 的手工别名入口。
- 使用固定 `/usr/bin/ssh -G <Host>` 预览 `LocalForward`、`RemoteForward` 和 `DynamicForward`。
- `Match exec` 执行前确认、10 秒超时和最多 4 个并发预览。
- 监听范围展示与可能暴露监听的风险确认。
- 批量导入只保存 Host 引用，不修改 SSH Config、不自动连接。
- 导入失败回滚、已有别名防重复和中英文界面。

- 应用版本：`0.3.2` 本地正式配置测试包。
- macOS：Version 26.5（Build 25F71）。

- 签名：ad-hoc；`codesign --verify --deep --strict` 通过。
- 测试日期：2026-07-18。

## 实机验收结果

### 发现与界面

- 导入面板正常打开并列出默认 SSH Config 中的明确 Host。
- 已导入 Host 显示“已导入”，复选框不可选择。
- 列表、错误区域和操作区可正常滚动及点击。
- 通配 Host 的具体别名入口默认折叠且明确标为可选，打开面板时不会聚焦输入框；不使用通配 Host 时无需填写。

### 预览与导入

- 使用一条脱敏本地转发配置预览，正确显示“本地转发”、监听端点和“回环监听”。
- 导入前本地配置共 8 条，导入后为 9 条；新增项状态为停止，未自动建立 SSH 连接。
- 新增 JSON 项只包含 `id`、模式、名称、`sshConfigName`、标签、收藏、手工顺序和自动重连开关，没有复制 HostName、用户、目标端点或转发指令。
- 新增项的自动重连保持默认关闭。
- 再次打开导入面板时，新导入的 Host 被识别为重复项且不可再次选择。

### 启动与只读边界

- 手工启动新导入配置后正常进入“运行中”；停止后没有残留受管 SSH 进程。
- 安装前、扫描后、预览后、导入后及最终包安装后，`~/.ssh/config` 的 SHA-256 始终一致。
- 覆盖安装最终测试包后，原有 9 条应用配置完整保留。
