本文记录 GitHub Issue #7“增加自动重连与网络、睡眠恢复”在 2026-07-18 完成的最终验收。

功能范围包括：

- 每条配置独立、默认关闭的自动重连开关。

- 认证失败、Host Key 失败、端口冲突和配置错误不自动重试；自动恢复期间的 `ssh -G` 瞬时超时继续按退避重试。

## 实机验收结果

在两条已明确开启自动重连的脱敏测试配置上完成真实 macOS 桌面验收。

### 退避与单连接约束

- 终止一条应用管理的 SSH 进程后，首档约 2 秒生成一个替代进程。
- 连续制造可恢复异常后，观察到 5、10、30 秒档位，没有高速循环。
- 另一条隧道在整个测试期间保持单一进程，不受目标隧道异常影响。
- 每次恢复只生成一个新 SSH 进程，没有并行重复进程。

### 等待阶段手工停止

- 把目标隧道推进到约 30 秒“等待重连”状态后点击“停止”。
- 跨过完整退避窗口后目标隧道没有再次生成进程。
- 另一条运行中的隧道保持正常。

### 睡眠与网络恢复

- 两条隧道运行时让 Mac 真正进入睡眠并在 10 秒后唤醒。
- 睡眠前进程退出后，唤醒只为每条配置生成一个替代进程；两个本地监听均恢复。
- 关闭当前默认 Wi-Fi 接口至少 10 秒后重新开启。
- 网络恢复后两条配置各生成一个替代进程，两个本地监听均恢复，没有重复启动。

### 不可重试错误

- 使用已被应用外 SSH 进程监听的本地端口启动配置。
- 应用显示“失败”和端口占用诊断，没有接管、终止外部进程或自动重试。
- 外部 SSH 进程保持不变。
