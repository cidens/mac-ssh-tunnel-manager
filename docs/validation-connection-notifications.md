## 实机验收结果

### 设置与权限

- 首次检查时连接通知默认关闭。
- 主动启用后才出现 macOS 通知权限请求；允许权限后设置持久化生效。
- 设置面板采用统一草稿语义：修改后“保存”生效并持久化，“取消”不落盘；重新启动应用后仍保持已保存状态。

### 连接详情与诊断复制

- 隧道卡片保持紧凑，仅通过信息按钮打开独立的“连接详情”面板。
- 详情正确显示状态变化时间、退出码、重试次数、下次重试时间、错误类别和已脱敏错误摘要。
- 复制结果包含应用版本、系统版本、架构、隧道模式、状态、状态变化时间、退出码、重试次数、下次重试时间和错误类别。
- 复制结果不包含配置名称、Host/IP、用户名、目标端口、私钥路径、完整 SSH 命令或原始 stderr。

### 通知周期与自动恢复

- 对一条已启用自动重连的脱敏测试隧道终止其受控 SSH 子进程。
- 每次连续故障周期仅出现一次失败通知；自动重连稳定后仅出现一次恢复通知。
- 面板保持打开、应用处于前台时，失败与恢复横幅均可见，并保留通知中心记录和提示音。
- 恢复通知在新 SSH 进程连续存活 2 秒后发送；重连过程中没有重复通知。
- 自动重连成功后隧道恢复为运行或监听状态。

### 用户主动操作

- 点击“停止”后隧道保持停止，不会自动重启。
- 用户主动停止不会发送连接失败或掉线通知。

## 验收结论

Issue #3 的自动化测试、正式配置构建、权限启用、前台通知、故障周期去重、自动恢复、主动停止、连接详情和诊断隐私边界均已通过，满足功能验收条件。
