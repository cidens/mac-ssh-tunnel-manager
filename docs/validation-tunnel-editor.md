# 隧道编辑器 UI 验收计划

## 范围

- 目标版本：`0.4.0` 开发版本。
- 需求文档：`docs/requirements-tunnel-editor.md`。
- 设计文档：`docs/design-tunnel-editor.md`。
- 本文在实现前记录验收合同；实际结果只能在对应证据取得后补充。

## 自动化验证

1. `swift test` 全量通过，现有安全、持久化、SSH 参数和迁移断言不修改语义。
2. `TunnelDraft` 可准确判断未修改与已修改状态。
3. 新建请求使用空草稿，编辑请求保留原配置和规则 UUID。
4. 新增规则仍受完整字段和 20 条上限约束。
5. 中英文资源键集合一致。
6. `git diff --check` 通过。

## 正式手工验收

### 主列表稳定性

1. 记录主面板当前滚动位置。
2. 分别打开新增和列表中部配置的编辑器。
3. 确认主卡片没有展开、跳位或出现空白；关闭后回到原滚动位置。

### 编辑器布局

1. 确认标题和底部取消/保存始终可见。
2. 在 1 条、3 条和 20 条规则下滚动到首尾。
3. 确认任何时刻只有一条规则展示完整字段。
4. 展开另一条规则，确认原规则收起且输入值保留。

### 新增、删除与排序

1. 当前规则未完整时“添加规则”不可用并显示原因。
2. 填写完整后新增，新规则自动展开并进入可视区域。
3. 第一条规则不能删除，其他规则可删除。
4. 上下移动后摘要和展开项身份保持正确，保存后顺序持久化。

### 取消保护

1. 未修改时取消立即关闭。
2. 修改名称或规则后取消，确认出现放弃提示。
3. 选择继续编辑，草稿保持。
4. 再次取消并放弃，主列表和磁盘配置保持原值。
5. 使用 Escape 重复检查相同行为。
6. 对连接组和 SSH Config 引用分别验证：未修改立即关闭，隐藏字段变化不产生误提示，实际可见字段变化必须提示。

### 保存与确认层级

1. 制造端口冲突，确认错误显示在 Sheet 内且草稿保留。
2. 修改危险监听，确认风险提示位于 Sheet 上方；取消后继续编辑。
3. 编辑运行中的连接组，确认统一重启提示位于 Sheet 上方；确认后 Sheet 关闭且连接组只保留一个新 SSH 进程。
4. 保存正常配置，确认 Sheet 关闭、主卡片摘要更新且主列表无跳跃。
5. 新建成功后确认主列表自动定位到新增卡片；编辑、取消和保存失败仍保留原滚动位置。
6. 保存成功、放弃修改和确认运行中重启后，确认只关闭 Sheet，主面板继续显示。

### 类型与双语

1. 新建时切换连接组与 SSH Config 引用，字段集合正确。
2. 编辑已保存配置时不出现类型切换入口。
3. 分别以英文和简体中文启动，确认标题、摘要、错误、取消确认和按钮无截断。

## 失败条件

- 主列表仍内联渲染完整表单。
- Sheet 打开或关闭造成主列表滚动位置明显变化。
- 长表单出现不可达字段、不可见操作按钮或空白区域。
- 风险确认或重启确认被 Sheet 遮挡。
- 取消可静默丢弃修改，或保存失败导致草稿丢失。
- 为通过测试而改变既有保存前安全拒绝语义。

## 回滚

若真实应用包出现菜单面板关闭、Sheet 无法操作、确认层级错误或数据丢失，停止合并并恢复原内联编辑视图。配置模型和磁盘文件不变，无需数据回滚。
