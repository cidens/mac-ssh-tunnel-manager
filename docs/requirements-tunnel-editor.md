# 隧道编辑器 UI 重构需求

## 摘要

- 目标：解决新增或编辑连接组时，主列表卡片高度剧烈变化造成的跳跃、错误滚动和空白区域，并降低多规则表单的视觉复杂度。
- 最终交付物：独立的新增/编辑 Sheet、固定高度的主列表卡片、单规则展开编辑、稳定的内部滚动和明确的取消/保存行为。
- 完成判定：主列表不再内联展开表单；包含 20 条规则的编辑器仍可操作；新增、编辑、风险确认、运行中重启和保存失败流程均通过自动化与真实应用包验收。

## 当前事实

- `TunnelMenuView` 的主列表使用 `ScrollView` 和 `LazyVStack`。
- 新建表单位于主滚动区，编辑表单位于 `TunnelRowView` 卡片内部。
- 展开编辑、增加规则、焦点变化和 `ScrollViewReader.scrollTo` 会同时改变懒加载列表中单行的高度与滚动位置。
- 连接组最多包含 20 条规则；当前界面同时完整展开所有规则字段。
- 设置、SSH Config 导入、配置导入导出和连接详情已经使用 Sheet，菜单面板具备维持模态交互的协调逻辑。
- 配置模型、schema v2、旧配置迁移、共享 SSH 进程及风险确认已经完成验收，本次不改变这些行为。

## 范围

### 范围内

- 新增和编辑配置改为独立 Sheet，不再改变主列表卡片高度。
- 编辑器使用固定标题区、内部滚动内容区和固定操作区。
- 连接组规则默认以摘要卡片显示，一次最多展开一条规则的详细字段。
- 新增规则后自动选中并展开新规则，只滚动编辑器内部。
- 新建配置允许选择“连接组”或“SSH Config 引用”；已保存配置仍不能切换类型。
- 取消存在未保存修改的编辑器时，明确询问是否放弃。
- 未修改判断只比较当前配置类型实际可编辑的字段，不受隐藏字段或控件初始化写回影响。
- 隧道显示名称按去除首尾空白、大小写不敏感方式保持唯一，重名时拒绝保存并保留草稿。
- 保存错误、运行中统一重启和危险监听确认必须显示在编辑器可见层级。
- 中英文界面、键盘操作和基本辅助功能标签同步更新。

### 范围外

- 不修改 `TunnelConfig`、`TunnelForwardRule` 或 JSON schema。
- 不修改 SSH 参数生成、进程生命周期、自动重连、自动启动、通知或登录项行为。
- 不修改配置导入导出、SSH Config 扫描或备份格式。
- 不新增多窗口同时编辑，也不允许同时打开多个编辑器。
- 不在本次引入完整的常驻“隧道管理”主窗口。

### 必须保持不变

- 保存前继续执行现有字段校验、端口冲突检查和 SSH Config 安全检查；不得为缩短等待而把既有拒绝断言改为启动时拒绝。
- 运行中的连接组保存前继续提示将统一重启。
- 风险确认继续绑定规则模式、监听地址和端口；修改端点后旧确认失效。
- 写盘失败时内存和磁盘配置保持原值。
- `.vscode/` 不进入提交。

## 功能需求

### R1 主列表稳定

- 主列表卡片只显示名称、摘要、状态和快捷操作。
- 点击“添加隧道”或“编辑”不得在主列表中插入或展开大型表单。
- 打开、编辑和关闭编辑器时，现有卡片的高度和顺序不得变化。

### R2 独立编辑器

- 新增和编辑使用同一编辑器组件，并通过模式决定标题、类型选择和保存动作。
- 编辑器内容区可独立纵向滚动；标题及“取消/保存”操作保持可见。
- 编辑器只允许单实例存在，关闭后才能打开另一个模态任务。

### R3 规则摘要与单项展开

- 每条规则的折叠状态显示启用状态、模式、监听端和目标端摘要。
- 同一时刻最多展开一条规则；展开另一条时收起原规则。
- 展开区保留模式、地址、端口、URL、启用、移动和删除能力。
- 第一条规则不可删除；其他规则可删除；上下移动边界与现有行为一致。

### R4 新增规则

- 当前规则未填写完整时继续禁用“添加规则”，并显示原因。
- 新增后自动展开新规则并滚动到编辑器可视区域。
- 一个连接组最多 20 条规则；达到上限时明确提示。
- 不得因连续点击生成多个空规则。

### R5 取消与关闭

- 未修改草稿时点击取消立即关闭。
- 草稿已修改时点击取消或按 Escape，显示“放弃未保存修改”确认。
- 选择继续编辑时保持所有输入和焦点上下文；选择放弃时不写盘、不重启隧道。

### R6 保存、错误和风险确认

- 保存按钮调用现有 `TunnelManager.addTunnel` 或 `updateTunnel`，不复制业务校验。
- 保存失败时编辑器保持打开，在固定操作区附近显示错误，输入不得丢失。
- 运行中编辑触发的统一重启确认和危险监听确认必须覆盖在编辑 Sheet 内，而不是被 Sheet 遮挡在主面板。
- 保存成功后关闭编辑器并刷新主列表；运行中更新按既有流程重启整个连接组。
- 新建成功后主列表滚动到新增卡片，使保存结果立即可见；编辑、取消和保存失败不改变主列表原滚动位置。

### R7 SSH Config 引用

- 新建时可选择 SSH Config 引用。
- SSH Config 引用只显示 Host 别名及通用字段，不显示连接组规则列表。
- 已保存引用的编辑器不显示配置类型切换入口。

## 非功能需求

- 编辑 20 条规则时界面持续响应，不出现主列表空白或跳到错误卡片。
- 编辑器规则容器不使用会因大幅高度变化产生回收空白的主列表懒加载结构。
- 控件具备可理解的中英文标签、帮助文本和 VoiceOver 名称。
- 不记录、上传或额外持久化编辑过程数据。

## 验收标准

| 编号 | 验收条件 | 成功证据 | 失败信号 |
| --- | --- | --- | --- |
| AC-01 | 新增和编辑不改变主卡片高度 | 打开与关闭 Sheet 前后卡片布局一致 | 表单仍在卡片内展开或列表跳位 |
| AC-02 | 长表单可操作 | 20 条规则可在编辑器内滚动到首尾 | 底部字段或按钮不可达、出现空白 |
| AC-03 | 单规则展开 | 任意时刻最多一条规则显示完整字段 | 多条详情同时展开导致高度失控 |
| AC-04 | 新增规则定位正确 | 新规则自动展开并进入可视区域 | 新规则隐藏在屏幕外或主列表滚动 |
| AC-05 | 取消保护 | 脏草稿取消时询问，放弃后磁盘不变 | 误保存、直接丢失或无法取消 |
| AC-06 | 保存失败可恢复 | 错误可见、Sheet 保持、输入保留 | Sheet 关闭或错误位于不可见层级 |
| AC-07 | 风险与重启确认可见 | 两类确认均显示在 Sheet 上方 | 确认出现在主面板后方 |
| AC-08 | 业务行为无回归 | 全量测试及共享进程手工抽查通过 | 配置格式、SSH 命令或启停行为变化 |
| AC-09 | 双语一致 | 中英文资源键集合一致并手工检查 | 缺键、截断或语义不一致 |

## 不算完成

- 只把整个内联表单包进另一个滚动区，但仍让主卡片改变高度。
- 只增加窗口高度，20 条规则仍同时展开。
- 自动化测试通过，但风险确认被 Sheet 遮挡或真实应用仍出现空白。
- 为避免保存等待而删除或延后既有 SSH Config 安全检查。
- 保存成功但取消、失败回滚或运行中重启路径没有验证。

## 回滚

- 本次不迁移数据；回滚只需恢复 `TunnelMenuView` 的内联呈现代码和相关本地化资源。
- 若独立 Sheet 导致菜单面板意外关闭、确认不可见或草稿丢失，停止合并并恢复原内联编辑，不修改用户配置文件。
