## 手工验收

### macOS 登录项

1. 安装版设置初始显示登录项关闭。
2. 开启“登录 macOS 时启动应用”并保存。
3. 重新打开设置，确认开关保持开启，或明确提示需要在系统设置中批准；界面不显示普通内部状态。
4. 关闭登录项并保存，确认逐连接自动启动选择没有被清除。

结果：通过。

### 逐连接自动启动

1. 选择一条使用回环监听地址的脱敏本地转发配置，在“自动化”分组中勾选“应用启动时连接”。
2. 确认 `tunnels.json` 只为该配置保存 `isAutoStartEnabled: true`，自动重连保持原值。
3. 退出应用，确认应用进程和测试端口均停止。
4. 手工重新打开应用，不点击主界面的“启动”。
5. 应用没有主动弹出面板，所选配置自动进入“已监听”；未选择的配置没有恢复运行。

结果：通过。

1. 展开中文界面的隧道编辑区。
2. 确认“应用启动时连接”和“断线后自动重连”归入同一个“自动化”分组。
3. 确认两项使用紧凑复选框，顺序与辅助说明清晰，未恢复旧版的大号开关布局。
4. 分别切换并保存两项，确认原有自动启动和自动重连行为不变。

结果：通过。

## 安全与兼容性结论

- 登录项只使用 macOS 官方 `SMAppService.mainApp`，没有创建 LaunchAgent。
- 登录启动和手工启动使用同一逐连接选择规则，不恢复全部上次运行项。
- 可能暴露监听地址的配置在新进程没有有效风险确认时不会静默自动启动。
- 单条自动启动失败不会阻塞其他已选配置。
- 登录项设置与逐连接选择相互独立，注销登录项不修改隧道配置。
- 未改变 SSH 配置文件、凭据、风险确认或已有运行时诊断的持久化边界。
