# 全局快捷键备用入口需求

## 文档状态

- 状态：已实现并完成核心场景验收，剩余多显示器和全屏空间验证
- 适用产品：SSH Tunnel Manager
- 适用平台：macOS 14 及以上
- 需求来源：菜单栏空间不足时，应用图标可能被 macOS 隐藏，用户需要不依赖菜单栏图标的备用入口。

## 背景

功能改造前，SSH Tunnel Manager 使用 SwiftUI `MenuBarExtra` 展示主界面，并通过 `LSUIElement=true` 隐藏 Dock 图标。当时应用没有全局快捷键，也没有独立的 Dock 或主窗口入口；菜单栏空间不足时，用户只能先腾出空间，或者从 Finder、Spotlight、Launchpad、终端重新定位应用。

本需求增加一个默认启用、可由用户修改的全局快捷键。只要应用仍在运行，即使菜单栏图标不可见，用户也可以通过快捷键展示并置前现有主界面。

## 目标与完成判定

### 目标

- 提供不依赖菜单栏图标的主界面备用入口。
- 菜单栏入口使用纯图标以减少宽度占用，悬浮时显示动态名称和运行数量。
- 允许用户录制、保存、重置或停用全局快捷键。
- 保存前检查输入有效性以及 macOS 可识别的快捷键冲突。
- 发生已确认冲突时给出明确提示，并保持原快捷键继续可用。
- 重启应用后恢复用户已经保存的快捷键设置。

### 完成判定

以下条件必须同时满足：

1. 应用运行且菜单栏图标不可见时，已注册的全局快捷键仍能打开主界面。
2. 用户可以在应用内录制其他组合键；通过校验且没有已确认冲突的组合键可以保存并立即生效。
3. 已确认冲突、非法组合或注册失败时不能覆盖现有设置，界面会说明原因。
4. 快捷键配置在正常退出和重新启动应用后保持不变。
5. 自动化测试和真实 macOS 手工验收覆盖正常路径、冲突路径、重启恢复及菜单栏图标不可见的场景。

## 立项时事实与约束

### 已验证事实

- 改造前应用入口定义在 `Sources/SSHTunnelManagerApp/SSHTunnelManagerApp.swift`，只有一个窗口样式的 `MenuBarExtra`。
- 改造前主界面为 `TunnelMenuView`，包含隧道列表、添加、编辑、启动、停止、刷新和退出入口，没有设置入口。
- 应用包通过 `scripts/build-app-bundle.sh` 写入 `LSUIElement=true`，因此正常运行时不显示 Dock 图标。
- 项目最低支持 macOS 14，使用 Swift 6 和 SwiftPM，当前没有第三方依赖。
- 隧道定义保存在 `~/Library/Application Support/ssh-tunnel-manager/tunnels.json`；全局快捷键不是隧道业务配置，不应写入该文件。
- macOS 可以提供已启用的系统级符号快捷键信息，并可以报告部分全局快捷键独占注册冲突，但不能完整枚举其他应用内部定义的命令快捷键。

### 实现约束

- 快捷键必须在其他应用位于前台时生效，不能只使用 SwiftUI 视图内的 `.keyboardShortcut`。
- 快捷键触发不能依赖模拟鼠标点击、固定屏幕坐标或辅助功能权限。
- 不得要求用户授予“辅助功能”或“输入监控”权限。
- 不得记录普通键盘输入。只有快捷键录制控件处于录制状态时才能读取下一次组合键。
- 新功能不能改变隧道配置格式、SSH 进程生命周期、退出清理和现有菜单栏点击行为。
- 新增界面文案必须同时提供英文和简体中文本地化；本文档仍只使用中文记录。

### 立项时待验证项（已完成）

改造前的 `MenuBarExtra` 没有项目内已验证的编程式展示入口。实现阶段验证后采用 `NSStatusItem + NSPanel`，全局事件稳定复用同一个 `TunnelMenuView` 和 `TunnelManager`，没有复制第二套隧道状态。

## 术语与冲突检测边界

- **当前快捷键**：已经持久化并正在使用的快捷键。
- **候选快捷键**：用户本次录制、尚未保存的快捷键。
- **已确认冲突**：候选快捷键与已启用的 macOS 系统级快捷键相同，或者向系统尝试独占注册候选快捷键时返回明确的占用结果。
- **潜在冲突**：其他应用使用了 macOS 无法枚举或未独占注册的相同组合。应用无法保证在保存前发现所有此类情况。
- **注册失败**：不是快捷键占用，但系统仍未能注册候选快捷键的情况。

产品文案和发布说明不得宣称可以检测“全部快捷键冲突”。界面应说明检测覆盖系统可识别的冲突；如果保存成功但实际按键没有生效，排障文档应引导用户更换组合，并检查其他应用的快捷键设置。

## 用户故事

1. 作为菜单栏项目很多的用户，我希望按一个固定组合键打开 SSH Tunnel Manager，以便图标被隐藏时仍能管理隧道。
2. 作为已经使用多个全局快捷键的用户，我希望修改默认组合，避免打乱现有操作习惯。
3. 作为配置快捷键的用户，我希望保存前得知系统能够确认的冲突，而不是保存一个确定无法工作的组合。
4. 作为升级用户，我希望失败的快捷键修改不会破坏原本可用的备用入口。

## 功能需求

### FR-01 默认快捷键与启用状态

- 默认快捷键为 `⌃⌥⌘T`（Control + Option + Command + T）。
- 首次安装以及从没有快捷键配置的旧版本升级时，全局快捷键默认配置为启用；没有已确认冲突且注册成功时立即生效。
- 应用启动后应尽早注册保存的快捷键，不要求用户先打开菜单栏界面。
- 如果默认快捷键或已保存快捷键在启动时发生冲突，应用继续运行，但该快捷键状态必须显示为“注册失败”，不能显示为可用。
- 全局快捷键只能在应用运行期间生效，不能用于启动尚未运行的应用。Finder、Spotlight、Launchpad 和 `open -a 'SSH Tunnel Manager'` 仍是启动应用的入口。

### FR-02 快捷键触发行为

- 当主界面不可见时，按下快捷键应在当前使用的桌面空间中展示主界面，并把它置于可交互状态。
- 当主界面已经可见时，再次按下快捷键应关闭主界面，形成明确的显示/关闭切换行为。
- 快捷键触发的界面必须复用现有 `TunnelMenuView` 及同一个 `TunnelManager` 实例，不能创建第二套隧道运行状态。
- 在多显示器环境中，界面应出现在鼠标指针所在的显示器和当前桌面空间，不得出现在用户不可见的其他桌面空间。
- 应用收到再次打开或重新激活请求时，例如对已运行应用执行 `open -a 'SSH Tunnel Manager'`，也应展示并置前主界面。该行为是快捷键启动注册失败时的恢复入口。
- 锁屏、登录窗口或系统安全输入阻止快捷键事件时，应用不做额外模拟输入或权限绕过。

### FR-03 设置入口

- 在现有主界面提供清晰的“设置”入口。
- 设置界面至少包含：
  - “启用全局快捷键”开关；
  - 当前或候选快捷键显示；
  - “录制快捷键”操作；
  - “恢复默认”操作；
  - “保存”和“取消”操作；
  - 当前注册状态以及错误提示。
- 打开设置界面时不得立即修改当前快捷键。只有保存成功后，候选快捷键才成为当前快捷键。
- “取消”应丢弃未保存的候选设置，当前快捷键保持不变。
- “恢复默认”只把候选值改为默认组合，仍需经过冲突检测和保存流程。
- 用户可以关闭全局快捷键。保存关闭状态后应注销当前快捷键，并明确提示备用入口已经停用。
- 启动注册失败时应提供“重试”操作；即使快捷键值没有变化，重试也必须重新执行系统冲突查询和注册，不要求用户先改成其他组合。

### FR-04 快捷键录制与输入校验

- 点击录制后，控件进入明确的录制状态，并提示用户按下新组合键。
- `Escape` 取消本次录制，不改变候选值。
- 录制期间应暂时抑制当前全局快捷键的展示或置前动作，避免该动作干扰组合键捕获和设置界面焦点。
- 一个有效组合必须包含：
  - 一个非修饰键；
  - 至少一个 `Control`、`Option` 或 `Command` 修饰键。
- 单独的修饰键、普通字母或数字、仅含 `Shift` 的组合均视为非法，不能保存。
- 不允许使用系统保留且无法可靠注册的单键，例如电源键、媒体键或纯功能修饰键。
- 保存的数据应包含稳定的物理键码和修饰键集合，界面显示为用户可读的 macOS 符号形式，例如 `⌃⌥⌘T`。
- 切换键盘布局后，应用应继续识别已保存的物理按键，并根据当前布局更新可读显示；不得因字符显示变化丢失配置。

### FR-05 保存前冲突检测

用户点击“保存”时，按以下顺序处理：

1. 校验启用状态和候选快捷键格式。
2. 候选值与当前值相同、启用状态未变化且运行时注册状态与配置一致时，不重复注册；如果配置为启用但当前注册失败，则按重试处理并继续后续检查。
3. 检查候选组合是否与应用自身保留的全局组合重复。
4. 检查候选组合是否与当前已启用的 macOS 系统级符号快捷键相同。
5. 在保留当前快捷键有效的前提下，向系统尝试独占注册候选快捷键。
6. 候选注册成功后持久化候选配置；持久化确认成功后，才切换事件处理并注销旧快捷键。
7. 候选注册或持久化任一步骤失败时，注销候选注册并回到修改前状态。

检测结果分为以下四类：

| 结果 | 界面行为 | 是否保存 | 旧快捷键 |
| --- | --- | --- | --- |
| 可以注册 | 显示保存成功和新组合 | 是 | 新组合生效后注销 |
| 已确认冲突 | 在录制控件附近显示冲突提示，并保留候选值供修改 | 否 | 继续有效 |
| 其他注册失败 | 显示可诊断错误和重试建议 | 否 | 继续有效 |
| 持久化失败 | 显示保存失败和重试建议，注销候选注册 | 否 | 继续有效 |

已确认冲突的推荐文案：

```text
快捷键“⌃⌥⌘T”已被 macOS 或其他应用占用，请更换组合后再保存。
```

其他注册失败的推荐文案：

```text
无法注册快捷键“⌃⌥⌘T”。原快捷键未改变，请重试或选择其他组合。
```

持久化失败的推荐文案：

```text
无法保存快捷键设置。原快捷键未改变，请重试。
```

由于系统无法可靠提供占用方名称，提示不得猜测或显示未经确认的应用名称。已确认冲突不得提供“仍然保存”或“强制覆盖”操作。

### FR-06 持久化与启动恢复

- 快捷键设置应与 `tunnels.json` 分离，保存在应用偏好设置中。
- 最少保存以下字段：配置版本、是否启用、物理键码、修饰键集合。
- 没有保存记录时使用 FR-01 定义的默认值。
- 配置缺字段、键码无效或修饰键集合无法解析时，应用不得崩溃；应回退到默认候选值，并在设置界面显示配置异常。
- 应用启动时如果保存的快捷键注册成功，设置界面显示“已启用”。
- 应用启动时如果注册失败，应保留用户保存值以便修改，显示“注册失败”，且不得静默换成其他快捷键。
- 覆盖安装、正常升级和应用重启不得清除用户设置。
- 偏好设置无法写入或写入后校验不一致时，应按 FR-05 回滚候选注册，不得把未持久化的新组合显示为已启用。

### FR-07 状态与可访问性

- 设置界面必须区分“已启用”“已停用”“录制中”“有未保存修改”“冲突”“注册失败”状态。
- 错误提示不能只依赖颜色，应同时包含文字和适当的可访问性描述。
- 快捷键录制控件、开关和按钮必须支持键盘导航及 VoiceOver 可读标签。
- 新增状态和错误文案必须接入 `AppStrings` 本地化体系。

## 非功能需求

### 兼容性

- 支持项目声明的 macOS 14 及以上版本。
- 兼容 Intel 和 Apple Silicon 构建目标；快捷键配置不得序列化与 CPU 架构相关的内存结构。
- 不新增 Accessibility 或 Input Monitoring 权限依赖。

### 稳定性

- 重复展示和置前主界面不得创建重复事件监听器或重复 `TunnelManager`。
- 多次保存快捷键后，进程内最多只能有一个用于打开主界面的有效快捷键。
- 应用正常退出或停用快捷键时应注销注册；异常退出后的系统注册不得残留。
- 快捷键触发不得阻塞主线程上的隧道状态刷新或 SSH 进程管理。

### 隐私与安全

- 快捷键配置仅保存在本机，不上传、不记录遥测。
- 不记录用户在录制状态之外按下的键。
- 日志如需记录注册失败，只记录错误类别和组合键，不记录连续键盘输入或其他应用信息。

## 范围外

- 使用快捷键启动尚未运行的应用。
- 登录时自动启动应用。
- 为每条隧道分别配置启动或停止快捷键。
- 自动修改或关闭 macOS、其他应用的冲突快捷键。
- 保证识别所有第三方应用内部、非独占或动态变化的快捷键。
- 云同步快捷键设置。
- 增加 Dock 常驻图标或增加命令行控制接口。

## 目标实现边界

以下结构用于明确职责，不限定必须采用某个第三方库：

- **快捷键模型与校验**：保存键码、修饰键、启用状态，提供纯逻辑校验和显示格式化。
- **系统快捷键注册器**：封装注册、注销、系统冲突查询和事件回调，允许在测试中替换为假实现。
- **快捷键设置存储**：独立于隧道配置存储，负责默认值、版本和异常配置恢复。
- **界面展示协调器**：接收全局快捷键事件，控制现有主界面的显示和置前。
- **设置界面**：只编辑候选状态，通过保存操作调用冲突检测和注册事务。

如果引入第三方 Swift Package，必须在实施前说明选择理由、许可证、维护状态、最低系统版本和冲突检测能力；引入依赖不能降低本需求定义的检测、回退和测试要求。

## 验收标准

| 编号 | 验收条件 | 验证方法 | 成功证据 | 失败信号 |
| --- | --- | --- | --- | --- |
| AC-01 | 菜单栏图标不可见时可打开主界面 | 制造菜单栏拥挤场景，在其他应用前台按默认快捷键 | 主界面出现在当前空间并可操作 | 无响应、要求先点击图标或界面出现在其他空间 |
| AC-02 | 快捷键切换主界面显示状态 | 主界面隐藏时触发一次，再触发一次 | 第一次展示并置前，第二次关闭；不产生重复窗口 | 无法展示、第二次仍保持显示或产生重复窗口 |
| AC-03 | 自定义快捷键保存后立即生效 | 录制无冲突组合并保存，分别测试旧组合和新组合 | 新组合生效，旧组合失效 | 两个组合同时生效或都不生效 |
| AC-04 | 自定义设置重启后恢复 | 保存新组合，正常退出并重新启动 | 设置显示保存值且新组合生效 | 回到默认值或显示值与实际注册不一致 |
| AC-05 | 系统快捷键冲突阻止保存 | 录制一个当前已启用的 macOS 系统快捷键并保存 | 显示冲突，新值未保存，旧组合继续生效 | 覆盖旧组合、无提示或抢占系统功能 |
| AC-06 | 其他进程独占冲突阻止保存 | 使用测试辅助进程独占注册候选组合，再在应用中保存 | 显示冲突，旧组合继续生效 | 保存成功但新组合不可用，或旧组合丢失 |
| AC-07 | 非冲突注册错误可恢复 | 使用假注册器注入一般注册错误 | 显示一般错误且可重试，旧组合继续生效 | 错误被当作成功或设置损坏 |
| AC-08 | 非法组合不能保存 | 依次录制无修饰键、仅 Shift、仅修饰键 | 每种情况均显示格式提示 | 任一非法组合被保存 |
| AC-09 | 录制不会触发当前快捷键动作 | 打开设置并录制与当前快捷键相同的组合 | 设置界面焦点稳定，候选值被捕获 | 展示协调器被触发、焦点异常或录制中断 |
| AC-10 | 停用和恢复默认行为正确 | 停用后保存，再执行恢复默认和保存 | 停用时无响应；恢复默认后默认组合生效 | 停用后仍响应或恢复默认绕过冲突检测 |
| AC-11 | 启动时冲突不导致崩溃且可以重试 | 保存组合后退出，由辅助进程占用该组合，再启动应用；随后退出辅助进程并点击重试 | 冲突期间应用正常运行并显示注册失败；冲突解除后原组合注册成功 | 应用崩溃、静默改键、错误显示为已启用或无修改时无法重试 |
| AC-12 | 不请求敏感权限 | 全新用户环境运行并完成全部快捷键操作 | 无辅助功能或输入监控授权提示 | 需要敏感权限才能使用 |

## 不算完成

- 只增加视图内快捷键，切换到其他应用后不能触发。
- 快捷键只能在菜单栏图标可见或主界面已经打开时使用。
- 可以录制并保存，但保存前没有实际查询或试注册候选组合。
- 冲突后新组合不可用，同时旧组合也被注销。
- 只检测一份硬编码的常见快捷键列表，却声称完成系统冲突检测。
- 自动选择另一个快捷键替代用户保存值，却没有告知用户。
- 通过监听所有键盘事件或要求辅助功能权限实现快捷键。
- 为快捷键另建一套主界面或 `TunnelManager`，导致隧道状态不一致。
- 单元测试通过，但没有验证菜单栏图标隐藏、其他应用前台、全屏或多显示器场景。

## 发布与回退

### 发布前

- 更新架构说明、README 使用方式、排障手册和中英文更新日志。
- 如果引入第三方依赖，完成许可证和分发产物检查。
- 使用安装脚本生成真实 `.app`，完成所有必须的手工验收。
- 覆盖升级测试必须确认旧版 `tunnels.json` 不发生变化。

### 中止条件

出现以下任一情况不得发布：

- 只能依赖辅助功能或输入监控权限实现。
- 不能稳定地从全局事件展示现有主界面。
- 冲突或注册失败会造成旧快捷键失效。
- 快捷键触发可能创建多个 `TunnelManager` 或重复管理 SSH 进程。
- 菜单栏图标点击、退出清理或隧道管理发生回归。

### 回退

- 代码回退时保留未知的快捷键偏好字段，不删除用户隧道配置。
- 旧版本不读取快捷键偏好，不应受新增配置影响。
- 如果新版本上线后快捷键注册造成严重兼容问题，可以在后续版本停止读取和注册该设置；不得以清空 `tunnels.json` 作为恢复手段。

## 风险与已确认决策

### 已知风险

- **冲突检测不完整**：macOS 不提供全部第三方应用快捷键清单。通过系统符号快捷键查询、独占试注册和诚实的界面说明降低风险。
- **`MenuBarExtra` 展示控制不确定**：如果无法稳定编程式打开，需要重构展示层。通过最小原型和保持单一 `TunnelMenuView`、单一 `TunnelManager` 限制改动范围。
- **多显示器和桌面空间差异**：菜单栏弹窗定位可能随系统版本和显示器布局变化，必须进行真实环境验收。
- **键盘布局变化**：物理键码与显示字符可能变化，需把存储和显示职责分离。

### 已确认产品决策

1. 默认组合使用 `⌃⌥⌘T`。
2. 首次安装和从没有快捷键配置的旧版本升级后，快捷键在没有已确认冲突且注册成功时默认启用。
3. 快捷键采用显示/关闭切换行为：主界面不可见时展示并置前，已经可见时再次触发则关闭。
